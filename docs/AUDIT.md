# Brutal audit: from taste memo to verifiable design system

Audit date: 2026-07-10. The original installed skill was copied unchanged before the rewrite and tested as a black box. The audit did not assume that length, confident prose, or a passing regex script meant design capability.

## Baseline verdict

The original package had useful art-direction instincts, but it was not release-grade as a reusable public skill.

| Probe | Baseline result | Why it mattered |
|---|---:|---|
| Contract tests | 0 / 6 | The package lacked compact routing, explicit invocation metadata, production graphics contracts, measurable QA, and clean redistribution boundaries. |
| `SKILL.md` size | 556 lines, 8,392 words, 58,953 bytes | The core file loaded too much repetitive guidance before task work could begin. |
| Whole package | about 30,462 words | A large reference corpus was being treated as capability even when much of it was repeated or unrelated. |
| Existing verifier | passed 5 prompts | It checked phrases and counts, not generated interfaces, browser behavior, provenance, accessibility, or graphics lifecycle. |

The baseline also redistributed scraped competitor-derived material without a clear reuse license, included an unrelated internal admin-app specification, encouraged a repeatable three-media formula, and did not define WebGL disposal, context-loss, reduced-motion, low-power, or real-browser evidence.

## Forward tests against the baseline

Two builds were commissioned from the old skill before rewriting it.

### Wind Before Sound

A fictional heritage pipe-organ conservation page proved that the original taste rules could produce a coherent niche-specific SVG experience. Browser QA caught and fixed a masked hero object, a 768 px overflow, clipped mobile copy, and an oversized stylesheet. It passed the local six-check suite and produced desktop, mobile, and interaction captures.

The result was good editorial web design, but it exposed the ceiling: the old skill had no material-simulation architecture, no GPU lifecycle contract, no typed design manifest, and no independent evidence verifier.

### مسار السكن

A fictional Arabic RTL emergency-tenancy advice page tested the opposite constraints: regulated subject matter, no invented proof, 320 px reflow, reduced motion, no JavaScript, no WebGL, keyboard flow, and explicit prototype boundaries. The page kept all critical meaning in HTML and generated no horizontal overflow in the recorded runs.

This test is intentionally retained as a counterexample to “Three.js everywhere.” A premium result may be typographic and code-native when 3D would add risk or obscure the user’s job.

## Independent red-team findings

The first rewrite was reviewed by an independent agent. It found that a valid-looking manifest could omit required objects and still score 100 because the evaluator had not actually executed its JSON Schema. It also showed that screenshot names, command strings, booleans, claims, and asset paths could be fabricated.

That review changed the release design:

- manifest schema validation is a real executable gate, not documentation;
- manifest lint is named honestly and cannot claim that a browser run happened;
- target-project verification resolves artifacts and hashes rather than trusting strings;
- generated, original-code, owned, and licensed assets have source-specific requirements;
- verified claims require sources and production cannot carry placeholders;
- drag interactions need a non-drag single-pointer alternative;
- budgets and measured evidence are separated;
- the open-source license and third-party policy are part of the package contract.

A second release review rejected the first green evidence set because a fixed control obscured content, Foldline's manifest described typography it did not render, verified claims lacked source types, the performance report mislabeled a long-task proxy as sufficient evidence, and keyboard/zoom coverage was incomplete. A later evidence review also found that stored reports were not semantically bound to the current build. Those findings added reserved control rails, implementation-accurate manifests, required claim source types, real three-run Lighthouse TBT gates, complete route control/CTA traversal, 200-percent text zoom and 320-pixel target clipping checks, strict preview ownership, declared bundle/media budget gates, and build-fingerprint verification. The release remained blocked until a final isolated-process visual review passed every retained state.

## Rewrite

The new package is a compact router with focused references. It excludes ordinary brochure sites and CRUD dashboards by default, while retaining an escape hatch for regulated, accessibility-first work where spectacular graphics would be inappropriate.

Its material vocabulary is engineered rather than decorative:

- fluid systems: procedural flow fields, refraction, pressure lenses, and bounded GPU/CPU simulation tiers;
- optical glass: physical transmission, thickness, index of refraction, internal layers, and legible DOM overlays;
- deformable surfaces: cloth/page meshes, Verlet or position-based constraints, pinned edges, pointer forces, and deterministic resting states;
- authored fallbacks: a complete static composition for reduced motion, disabled graphics, asset failure, and low-power devices.

## Evidence boundary

No finite test can prove that a design is “the best in the world” or has never existed before. DEMETA instead makes a narrower, falsifiable promise: reject known template signatures, record the selected design fingerprint, compare distinct candidates, inspect rendered desktop/mobile results, and keep human aesthetic review advisory and explicit.

The showcase is fictional. It contains no customer results, awards, testimonials, certifications, real devices, real dive records, or legal advice.
